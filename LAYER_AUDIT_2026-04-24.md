# S.O.N Layer Audit — 2026-04-24

## Summary
- Total layers: 13
- WORKING: 4 (SEA, WAR, RSS, CAM)
- BROKEN_SHAPE: 6 (SAT, MIL, AIR, HOT, SOC, GPS)
- KEY_GATED: 0 (all key-gated adapters are *also* shape-broken; classified below under BROKEN_SHAPE where the shape is the blocker even with the key)
- DEAD_SOURCE: 0 (Bluesky/Nitter do return posts, but shape-broken downstream)
- NO_ADAPTER: 3 (NET, ASP, plus effectively SAT has no TLE-carrying adapter)

Note: I use BROKEN_SHAPE whenever the render function reads a key that synthesize() demonstrably never writes — even if the underlying adapter also requires a key. The BROKEN_SHAPE fix is strictly required before the key helps anything.

## Fix Priority

Ranked by: impact × smallness-of-diff, BROKEN_SHAPE before KEY_GATED, user-visible wins first.

1. **AIR (Commercial Flights) — BROKEN_SHAPE, medium diff.** Two-step fix inside `apis/sources/opensky.mjs` + `dashboard/inject.mjs`: (a) in `opensky.mjs::briefing()`, preserve the first ~500 `states` per hotspot as a top-level `states` array with `{icao24, callsign, country, lon, lat, alt}` objects; (b) in `synthesize()`, add `const opensky = { states: (data.sources.OpenSky?.states || []) }; V2.opensky = opensky;`. Unlocks the highest-density layer on the dashboard (~hundreds of points globally) with no key required. Biggest visible win.

2. **MIL (Military Flights) — BROKEN_SHAPE, tiny diff, but key-gated underneath.** In `synthesize()`, add: `const adsb = { militaryAircraft: data.sources['ADS-B']?.militaryAircraft || [] }; V2.adsb = adsb;`. Without `ADSB_API_KEY` this yields 0 points (no worse than today) but the instant the key is added, the layer lights up. One-line patch.

3. **SAT (Satellite Passes) — BROKEN_SHAPE, small diff in `space.mjs`.** `apis/sources/space.mjs::getRecentLaunches()` and `getStationData()` discard `TLE_LINE1`/`TLE_LINE2`. Add `tle1: sat.TLE_LINE1, tle2: sat.TLE_LINE2` to both mappers, and in `synthesize()` promote them to `V2.space.satellites = [...recentLaunches, ...spaceStations].filter(s => s.tle1 && s.tle2)`. SatelliteTracker then renders animated orbits. Keyless, globally visible, very cinematic — high-impact UI win.

4. **HOT (Thermal Anomalies) — shape is actually correct; blocker is `FIRMS_MAP_KEY`.** Register at firms.modaps.eosdis.nasa.gov (free). Once set, existing synthesize code produces `firms.detections` correctly.

5. **SOC (Social Signals) — heuristic-limited.** Short-term: broaden the `geoKeywords` map (already large) and/or fall through to geo-tagging via the `bucket` (e.g. "Iran war" bucket → Iran coords). Longer-term: Nitter mirrors need re-curation since most public mirrors are 403 in 2026. Not a quick win.

6. **GPS (GPS Jamming) — verify or implement the `/api/proxy/gpsjam` server route in `dashboard/server.mjs`.** If missing, add a small proxy to `https://gpsjam.org/data/<today>.json`. No adapter file needed; route-level fix.

7. **NET (Internet Blackouts) — NO_ADAPTER.** Need new `apis/sources/cloudflare_radar.mjs` hitting `https://api.cloudflare.com/client/v4/radar/annotations/outages` (requires `CLOUDFLARE_API_TOKEN`). Medium effort; low frequency of content.

8. **ASP (Airspace Closures) — NO_ADAPTER.** NOTAM data is awkward (FAA SWIM requires account). Consider a lightweight adapter hitting `https://www.notams.faa.gov/common/notam.html` scrape or skip until a paid source exists. Lowest priority.
